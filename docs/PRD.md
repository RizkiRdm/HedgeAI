# PRD: CryptoHedgeAI Crew (v1.3)

## 1. Project Summary
Development of an autonomous quant trading agent using CrewAI for altcoin discovery and spot execution. The system leverages DuckDB for high-speed local data analysis and OpenClaw for remote command and control.

## 2. Problem Statement
- **Analysis Paralysis**: Manually scanning 500+ new altcoin pairs daily is impossible.
- **Data Latency**: Traditional API-only bots are too slow for breakout momentum.
- **Risk Discipline**: Human traders often ignore stop-losses; the bot must be ruthlessly algorithmic.

## 3. Success Metrics (Measurable)
- **Efficiency**: Fetch and process the 28 core metrics for Top 300 coins in **< 8 seconds**.
- **Data Integrity**: DuckDB cache hit rate **> 80%** for non-price metrics.
- **Financial**: Maintain a **Sharpe Ratio > 1.2** during the 30-day trial.
- **Sustainability**: `ops_wallet` successfully collects fees from >90% of winning trades.

## 4. Core Capabilities (MVP)
- **DuckDB Analytical Engine**: Replacing standard JSON/CSV with DuckDB for lightning-fast metric processing and caching.
- **Tiered Data Fetching**: Smart logic to differentiate between "Real-time Data" (Price) and "Stale-tolerant Data" (Holders, Github Commits).
- **Risk Guardian Veto**: A hard-coded requirement for the Risk Guardian to approve any trade signal generated by the Strategist.
- **Autonomous Profit Tax**: Automated logic to skim 0.5% profit to the operations wallet upon position closure.

## 5. System Architecture & User Flow
1. **Trigger**: Cron job wakes up the **Overseer**.
2. **Data Ingestion**: **Data Oracle** checks DuckDB. If cache is expired, fetches new data and updates DuckDB.
3. **Screening**: Filter coins based on liquidity and security (taxes/honeypot).
4. **Quant Brain**: **Strategist** runs technical analysis on filtered coins.
5. **Veto/Check**: **Risk Guardian** verifies portfolio correlation and drawdown limits.
6. **Execution**: **Trader** executes via CCXT.
7. **Post-Trade**: **Accountant** logs to DuckDB and handles profit sharing.

## 6. Implementation Notes (DuckDB)
- Use `.duckdb` persistent file.
- Tables: `market_cache`, `trade_history`, `agent_logs`, `performance_metrics`.
- Advantage: Allows the **Evaluator Auditor** to run complex SQL queries (e.g., "Win rate per category") instantly.

## 7. Non-Goals (MVP)
- No leverage/margin trading.
- No GUI/Frontend (All interactions via OpenClaw/Markdown reports).
- No support for multiple trading accounts.
